# Demo Walkthrough Checklist

Use this checklist to deliver the same demo every time.

## Before The Call

- Confirm the app runs locally.
- Confirm demo mode is enabled if you need the reset button.
- Reset demo data.
- Open the Dashboard tab.
- Keep `sales_assets/objection_handling.md` nearby.
- Know the buyer role: practice owner, office manager, or front-desk lead.

## Opening

- State that this is a lightweight inquiry and follow-up dashboard.
- Clarify that it does not replace the EHR, scheduler, billing system, or patient portal.
- Explain that all visible demo data is fake.

## Dashboard

- Start with Estimated Treatment Value.
- Point out Follow-Ups Needed and Overdue Follow-Ups.
- Explain Active Patients and Conversion Rate in plain language.
- Mention Top Inquiry Source.
- Keep charts optional unless the buyer asks for more detail.

## Patient Inquiries

- Show how a new patient inquiry is captured.
- Explain Requested Service, Inquiry Source, Status, Estimated Treatment Value, and Next Follow-Up Date.
- Show how follow-up updates stay connected to the inquiry.

## Weekly Summary

- Show the owner-ready weekly readout.
- Explain that the goal is a quick practice performance review, not a complex report.
- Show the Practice Performance Snapshot downloads if useful.

## Export

- Show patient-friendly CSV exports.
- Explain that exports support review, backup, and handoff.

## Close

- Ask whether this would help the practice avoid missed follow-ups.
- Ask where it would fit into the current front-desk workflow.
- Ask what would need to be true for a paid pilot.
- Log objections and next steps in `PROJECT_OS/outreach/outreach_tracker.csv`.
