# Proposal Workflow

Use this workflow to reduce friction between demo completion and signed client onboarding.

```text
Discovery Call -> Demo -> Proposal -> Decision -> Onboarding -> Go Live
```

## 1. Discovery Call

Goal: understand fit before proposing.

Actions:

- Ask about inquiry volume.
- Ask about current tracking method.
- Ask about follow-up ownership.
- Ask about weekly reporting.
- Identify decision-maker.
- Score the practice with the qualification scorecard.

Exit criteria:

- Prospect is at least a Possible Fit.
- Main pain is clear.
- Next step is a demo or no-fit decision.

## 2. Demo

Goal: show the inquiry and follow-up workflow clearly.

Actions:

- Start with the missed follow-up problem.
- Show Practice Performance Dashboard.
- Show one inquiry workflow.
- Show follow-up update.
- Show Weekly Summary.
- Show exports.
- Use ROI calculator only with conservative language.

Exit criteria:

- Prospect understands fit and limits.
- Main objection is known.
- Proposal package is likely.

## 3. Proposal

Goal: send a clear proposal using current pricing only.

Actions:

- Select Starter Demo Setup, Practice Launch, or Workflow Polish.
- Tailor current practice challenges.
- Confirm implementation timeline.
- Include training and support expectations.
- Send proposal and follow-up date.

Exit criteria:

- Proposal sent.
- Follow-up date scheduled.

## 4. Decision

Goal: reach a clear yes, no, or later.

Actions:

- Answer objections honestly.
- Avoid adding features to close the deal.
- Confirm selected package if accepted.
- Log outcome.

Exit criteria:

- Closed Won, Closed Lost, or timed follow-up.

## 5. Onboarding

Goal: reduce uncertainty immediately after acceptance.

Actions:

- Send one-page client onboarding packet.
- Confirm main contact.
- Confirm training attendees.
- Confirm data import needs.
- Confirm Day 1 setup.

Exit criteria:

- Client knows what happens next.

## 6. Go Live

Goal: start the operating workflow.

Actions:

- Confirm access.
- Confirm data appears.
- Train daily and weekly workflow.
- Confirm support process.
- Schedule first monthly review.

Exit criteria:

- Client can use the dashboard and knows the next review date.
