# Training Session Script

Use this for a 30-minute first-client training session with the practice owner, office manager, or front desk.

## 30-Minute Training Agenda

```text
0-3 minutes: Set expectations and confirm this is not an EHR, scheduler, billing system, or insurance tool.
3-8 minutes: Dashboard walkthrough.
8-14 minutes: Patient inquiry workflow.
14-19 minutes: Follow-up workflow.
19-24 minutes: Weekly review workflow.
24-27 minutes: Export workflow.
27-30 minutes: Q&A and next steps.
```

## Opening

Start with the business purpose:

```text
This dashboard helps the practice see patient inquiries in one place, review who needs follow-up, and understand weekly inquiry activity. It does not replace your EHR, scheduler, billing software, or insurance workflow.
```

Confirm the practice workflow owner:

- Who adds new inquiries?
- Who updates status?
- Who sets next follow-up dates?
- Who reviews the Weekly Summary?

## Dashboard Walkthrough

Show the Practice Dashboard first.

Explain:

- Total Patient Inquiries: all saved inquiries.
- New This Week: inquiries created since Monday.
- Follow-Ups Needed: inquiries marked for follow-up or due today/past due.
- Overdue Follow-Ups: follow-ups that should already have happened.
- Active Patients: inquiries that became active patients.
- Estimated Treatment Value: estimated value for inquiries not marked Lost.
- Inquiry-to-Patient Rate: share of inquiries that became Active Patients.
- Top Inquiry Source: source with the most inquiries.

Training line:

```text
If you only check one screen each week, start here. It shows whether follow-ups are slipping and whether inquiries are turning into active patients.
```

## Inquiry Workflow

Walk through adding a fake inquiry.

Steps:

1. Open Patient Inquiries.
2. Enter patient name, phone, email, requested service, source, status, estimated value, notes, and next follow-up date.
3. Submit the inquiry.
4. Confirm the inquiry appears in the table.
5. Return to Dashboard and show that counts update.

Practice habit:

```text
Every inquiry should have a status and, if it needs action, a next follow-up date.
```

## Follow-Up Workflow

Show how to update one inquiry.

Steps:

1. Select an inquiry.
2. Review notes.
3. Update status.
4. Set or change next follow-up date.
5. Save the update.
6. Confirm overdue or due-today follow-ups are visible.

Recommended daily routine:

- Review Follow-Ups Needed.
- Handle overdue follow-ups first.
- Update status immediately after a call or message.
- Keep notes short and useful.

## Weekly Review Workflow

Open Weekly Summary.

Explain:

- Review total inquiries.
- Review new inquiries this week.
- Review Active Patients.
- Review Follow-Ups Needed and Overdue Follow-Ups.
- Review Estimated Treatment Value.
- Review top inquiry source.

Weekly owner question:

```text
What needs attention before next week starts?
```

## Export Workflow

Open Export.

Show:

- Patient inquiry CSV export.
- Follow-up CSV export.
- Snapshot or weekly summary export if needed.

Explain:

```text
Exports are useful for backup, review, and handoff. They are not a replacement for production privacy and storage planning.
```

## Q&A Section

Use these prompts:

- What part of the workflow feels easiest?
- What part feels like extra work?
- Who should own daily updates?
- What should be reviewed weekly?
- What would make the dashboard more useful without adding complexity?

## Close

End with:

```text
For the first week, focus on one habit: every inquiry gets a status and every follow-up gets a date.
```
