# Chiropractic Business OS Roadmap

This roadmap tracks product and service growth without changing the current database schema or adding unnecessary complexity.

## Current Stage: Outreach-Ready Demo

- Chiropractic-specific Streamlit app
- SQLite patient inquiry database
- CSV-backed fake sample data
- KPI dashboard
- Weekly Practice Summary
- Practice Performance Snapshot
- Patient-friendly CSV exports
- Demo reset mode
- Sales collateral, screenshots, and silent demo video

## Next 30 Days

- Use the current outreach package for chiropractor discovery.
- Collect feedback from at least 5 practice owners or office managers.
- Track objections, must-have workflow gaps, and pricing reactions.
- Improve sales materials based on real responses.
- Avoid adding EHR, scheduling, payments, or authentication until a paid client requires it.

## Next 60 Days

- Convert the strongest demo feedback into a narrow paid pilot scope.
- Create one client-specific deployment using fake or approved non-sensitive data.
- Tighten onboarding and handoff documents.
- Validate whether the Starter Demo Setup, Practice Launch, and Workflow Polish tiers match buyer expectations.

## Next 90 Days

- Package a repeatable deployment workflow.
- Build one public-facing portfolio case study.
- Decide whether the next technical investment should be persistence, access control, or practice-specific workflow customization.
- Keep the MVP focused on inquiry follow-up visibility and practice-owner trust.

## Deferred Until Paid Need Exists

- Authentication
- Payments
- Scheduling/calendar integration
- EHR integration
- Insurance billing automation
- AI features
- HIPAA-grade production compliance package
